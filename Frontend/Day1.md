#### <script>에서 한글 깨짐 현상
+ IE에서 `innerText`을 사용하여 파일 이름을 넣는 방식으로 코딩을 했는데 여기서 문제가 생겼다.
+ `innerText`을 사용하여 `url` 인덱스 값에 따라 해당 파일 이름을 넣어주었는데 그렇게 넣어준 한글로된 파일명이 깨지는 현상이 발생했다.
  + 예) 깃허브 [ㅎㅏㄴㄱㅡㄹ ㄲㅐㅈㅣㅁ]
+ 처음에는 인코딩 문제인 줄 알고 `document.charset`을 콘솔로 찍어봤는데 `UTF-8`이라고 잘 나와서 인코딩 문제는 아니였다.
+ 두 번째로는 혹시 IE여서 `innerText`가 안 먹히는건가?? 라고 생각했는데 오히려 `innerHTML` 보다 `innerText`가 더 지원이 잘 된다고 하였다.
+ 그러나 혹시 몰라서 `innerHTML`로 바꿔서 해봤는데 역시나 똑같은 문제가 발생했다.
+ 다른 방법으로는 `html`에는 각각의 파일명을 하드코딩 해놓고 `display:none;`을 시킨 뒤 if문을 사용하여 인덱스를 각각 조건문을 줘서 해당 인덱스에 `display:block;` 스타일을 적용하였다.
+ 콘솔로 `innerText`와 `innerHTML`을 찍어봤는데 애초에 파일 리스트를 배열로 만들어 놓은 곳에서 들어올 때부터 한글이 깨져서 들어왔다.
